### 1. 新增管理员账号 管理员可以看到自己名下所有人的任务

每个人登录后，默认就是自己的管理员。相当于一个租户。
管理员 可以管理用户。可以邀请邮箱加入租户，邀请时 要输入邮箱。和用户类型 有普通用户和管理员。如果是管理员 那也可以获得同样的权限。
普通用户自动看到自己的数据。不能看到别人的。但是管理员能看到租户下所有的数据。

### 2. CloakBrowser cookie 云端加密存储

然后创建任务时 就可以从云端读取 cookie。创建后点击开始运行任务 就会把 cookie 传给浏览器、浏览器解密后发给本地 Agent程序。程序再使用 cookie 登录平台。还有当一个任务运行中。如果其它用户 想用这个cookie 登录平台。就会提示用户当前账号正在被谁使用。
管理员可以在 cookie 中管理这些数据、普通用户可以新增 cookie。也可以删除自己创建的 cookie。

重点 cookie 创建时必须通过加密存储。只有在开始任务时 才会从云端读取 cookie并解密发送到本地 Agent程序。
要实现的是 所有人都不能解密 cookie。该租户下的人才能解密。至于使用什么加密方式。你自己想。我希望的是即使数据库被泄露。也不能解密 cookie。
cookie不一定是json、也有可能是一个文件。这个你自己考虑。应该通过CloakBrowser搜一下。

通过该功能可以实现 账号共享。而不是一个账号只能在一个设备上登录。

类比一下：

    你把一份文件放到保险柜里（AES 加密 cookie）
    保险柜密码 SK 是随机生成的
    你用张三的锁把密码锁起来（张三公钥加密 SK）
    你用李四的锁也把密码锁起来（李四公钥加密 SK）

    张三用自己的钥匙开锁 → 拿到密码 SK → 打开保险柜
    李四用自己的钥匙开锁 → 拿到密码 SK → 打开保险柜

    张三和李四不需要互相知道对方的钥匙
    管理员上传时只需要指定"分享给谁"，系统自动完成锁的操作

实际操作流程：

    管理员上传 cookie
      → 系统自动生成随机密码 SK
      → 系统用管理员的公钥加密 SK（管理员自己能用）
      → 管理员选择"分享给李四"
      → 系统用李四的公钥也加密一份 SK（李四也能用了）

    后续：
      管理员运行任务 → Agent 用自己的私钥解开 SK → 解密 cookie
      李四运行任务 → Agent 用自己的私钥解开 SK → 解密 cookie

用户界面上看到的就是：上传 cookie → 勾选分享给谁 → 完成。所有密钥操作后台自动完成

### 3. CloakBrowser 关闭时 应该要主动上传 cookie带上cookie id、同时删除本地的 cookie。这样能实现cookie 刷新。
