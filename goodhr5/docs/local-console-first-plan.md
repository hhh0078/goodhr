# GoodHR 本地控制台长期方案

本文档记录 GoodHR 长期架构改造方案：入口改成本地程序，用户下载安装包后从本地控制台网页登录和操作任务；云端只负责登录、会员校验、配置和更新。

## 一、目标

把 GoodHR 改成“本地重、云端轻”：

- 本地程序提供控制台网页、任务执行、本地数据库、浏览器控制、候选人数据、截图、下载文件和日志。
- 云端只提供验证码登录、会员校验、前端包更新、平台规则包更新、AI/系统配置。
- 用户不需要先打开官网后台，启动本地程序后直接打开本地控制台。

## 二、用户流程

1. 用户下载安装包。
2. 安装时选择安装目录。
3. 所有数据都放在安装目录下。
4. 打开本地程序后执行初始化。
5. 本地程序检查控制台前端包是否最新。
6. 如果不是最新，下载最新前端包。
7. 初始化成功后，默认浏览器打开本地控制台：

```text
http://127.0.0.1:9001/
```

8. 用户在本地控制台中通过云端验证码登录。
9. 登录成功后，任务、日志、候选人、截图等接口都连接本地程序。
10. 开始任务前，本地程序向云端校验会员剩余时间。
11. 校验通过后，本地执行主流程。

## 三、安装目录结构

第一阶段优先 Windows，默认不安装到 `Program Files`，避免权限问题。推荐默认安装目录：

```text
%LOCALAPPDATA%\GoodHR
```

用户也可以选择其他目录，例如：

```text
D:\GoodHR
```

目录结构：

```text
GoodHR/
  GoodHRLocalAgent.exe
  data/
    goodhr_local.db
    profiles/
    downloads/
    screenshots/
    browser/
    rules/
  frontend/
    current/
    releases/
  config/
    settings.json
```

## 四、本地程序职责

本地程序负责：

- 启动本地 HTTP 服务。
- 托管本地控制台前端页面。
- 初始化 SQLite。
- 保存任务、日志、候选人、截图索引和下载记录。
- 管理浏览器 profile 和 cookie。
- 执行招聘平台主流程。
- 检查并更新前端包。
- 检查并更新平台规则包。
- 开始任务前请求云端会员校验。

本地程序不负责：

- 用户密码保存。
- 会员状态最终判断。
- 云端套餐、团队、邀请、支付等管理。

## 五、云端职责

云端保留：

- 验证码登录。
- 用户身份。
- 会员剩余时间。
- AI 配置。
- 系统配置。
- 前端包 manifest。
- 平台规则包 manifest。

云端不再保存新任务的：

- 候选人详情。
- 任务日志。
- 截图。
- 下载文件。
- 平台 cookie/profile。

## 六、本地控制台前端

本地 FastAPI 同时提供 API 和静态页面。

访问入口：

```text
GET /
```

返回本地前端 `index.html`。

前端路由刷新时也回退到同一个 HTML，避免 404。

控制台内部 API 分两类：

- 本地 API：任务、日志、候选人、截图、下载、设置。
- 云端 API：验证码登录、会员校验、更新包、规则包、AI 配置。

## 七、前端包更新机制

云端提供 manifest：

```text
GET https://goodhr5.58it.cn/agent-console/manifest.json
```

示例：

```json
{
  "version": "2026.06.05",
  "url": "https://goodhr5.58it.cn/agent-console/console-20260605.zip",
  "sha256": "..."
}
```

本地启动时：

1. 读取本地前端包版本。
2. 请求云端 manifest。
3. 如果本地没有前端包，下载。
4. 如果云端版本更新，下载。
5. 校验 sha256。
6. 解压到 `frontend/releases/{version}`。
7. 切换 `frontend/current`。
8. 下载失败或校验失败时继续使用旧版本。

## 八、平台规则包更新机制

第一阶段只做 JSON 规则包，不远程执行 Python 代码。

规则包包含：

- 平台入口地址。
- 页面判断规则。
- 选择器。
- 点击动作。
- 字段提取规则。
- 打招呼按钮规则。
- 详情页规则。

本地启动和任务开始前都可以检查规则包更新。

规则包失败处理：

- hash 不一致：拒绝启用。
- 新规则运行失败：自动回滚到上一个可用版本。
- 云端不可用：继续使用本地旧规则。

## 九、本地 SQLite

SQLite 文件：

```text
data/goodhr_local.db
```

第一阶段表：

- `local_tasks`：任务。
- `local_task_logs`：任务日志。
- `local_candidates`：候选人。
- `local_settings`：本地设置。
- `local_rule_versions`：规则包版本。

后续可扩展：

- `local_positions`：岗位模板快照。
- `local_downloads`：下载文件索引。
- `local_screenshots`：截图索引。

## 十一、验收标准

- 安装时可以选择安装目录。
- 首次启动会初始化目录和数据库。
- 本地程序能自动检查并下载控制台前端包。
- 访问 `http://127.0.0.1:9001/` 能打开控制台。
- 控制台可以通过云端验证码登录。
- 登录后任务列表读取本地数据。
- 开始任务前必须联网校验会员。
- 任务日志、候选人、截图全部写本地。
- 云端 CPU 不再随任务执行步骤上涨。
- 前端包或规则包更新失败时，不影响旧版本继续使用。
