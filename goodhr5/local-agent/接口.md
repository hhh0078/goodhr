# GoodHR5 Local Agent 接口清单

本文档整理 `goodhr5/local-agent` 当前对外 HTTP API，便于前端和云端联调。

## 通用说明

- 基础地址示例：`http://127.0.0.1:9001`
- 请求体格式：默认 `application/json`
- 成功返回：通常为 `{"ok": true, ...}`
- 失败返回：
  - 业务错误：`4xx` + JSON 错误信息
  - 未捕获异常：`500 Internal Server Error`

## 健康检查

- `GET /health`
- 作用：检查本地程序是否在线，返回机器码、公钥、端口、绑定账号。

## 云端账号与 WS

- `POST /api/v1/session/bind-cloud-user`
- 作用：绑定云端用户到本地程序。
- 请求字段：`cloud_user_id`, `cloud_email`, `agent_token`, `public_key`
- `POST /api/v1/ws/connect`
- 作用：本地主动连接云端 WebSocket。
- 请求字段：`cloud_ws_url`, `token`
- `GET /api/v1/ws/status`
- 作用：查询本地到云端 WS 的连接状态。
- `POST /api/v1/ws/disconnect`
- 作用：主动断开当前 WS 连接。
- `POST /api/v1/tasks/{task_id}/start-ws`
- 作用：建立任务级 WS，通知云端开始任务。
- 请求字段：`cloud_api_base`, `cloud_ws_url`, `token`
- `POST /api/v1/tasks/{task_id}/stop-ws`
- 作用：停止云端任务，并在无活跃任务时断开 WS。
- 请求字段：`cloud_api_base`, `token`

## Profile 管理

- `GET /api/v1/profiles`
- 作用：查询本地 profile 列表。
- 可选参数：`platform_id`
- `POST /api/v1/profiles`
- 作用：创建 profile 元数据。
- `DELETE /api/v1/profiles/{profile_id}`
- 作用：删除 profile 元数据。

## 本地任务数据

- `POST /api/v1/tasks/init`
- 作用：初始化本地任务目录。
- `GET /api/v1/tasks/{task_id}/candidates`
- 作用：读取本地候选人列表。
- `POST /api/v1/tasks/{task_id}/candidates`
- 作用：写入候选人记录。
- `DELETE /api/v1/tasks/{task_id}/candidates/{candidate_id}`
- 作用：删除候选人记录。
- `GET /api/v1/tasks/{task_id}/screenshots`
- 作用：查询截图列表。当前版本默认不再长期保存截图，该接口主要保留兼容能力。
- `GET /api/v1/tasks/{task_id}/screenshots/{filename}`
- 作用：下载单张截图。
- `DELETE /api/v1/tasks/{task_id}/screenshots/{filename}`
- 作用：删除单张截图。
- `POST /api/v1/tasks/{task_id}/ocr`
- 作用：保存 OCR 文本到本地任务目录。

## 浏览器控制

- `POST /api/v1/browser/start`
- 作用：启动浏览器。
- 请求字段：
  - `persistent`: 是否持久化
  - `user_data_dir`: 账号目录名（会映射到 `local-agent/cookies/{目录名}`）
  - `headless`: 是否无头
  - `humanize`: 是否仿人类
  - `proxy`: 代理地址
  - `cookies`: 可选，数组；启动后立即注入到当前浏览器上下文
- `POST /api/v1/browser/stop`
- 作用：关闭浏览器。
- `GET /api/v1/browser/status`
- 作用：查询浏览器运行状态。
- `POST /api/v1/cookie-sync/config`
- 作用：兼容旧前端调用；关闭浏览器自动回传 cookie 已停用，不再上传 cookie。

## 页面操作

- `POST /api/v1/page/open`
- 作用：打开指定 URL 并注册为默认页面。
- 请求字段：`url`, `timeout`, `user_data_dir`(可选), `persistent`(可选), `headless`(可选), `humanize`(可选), `proxy`(可选), `cookies`(可选，数组，传入时先注入再导航)
- 注意：必须先调用 `/api/v1/browser/start`；未启动会返回 `400`。
  - 如果传入 `user_data_dir`，接口会自动按该账号目录切换浏览器再打开页面，不需要先单独调用 `browser/start`。
- `POST /api/v1/page/close-by-url`
- 作用：关闭所有 URL 包含指定文本的页面。
- 请求字段：
  - `url_contains`：必填；例如传入 `333333`，会关闭所有 URL 中包含 `333333` 的页面
- 返回字段：
  - `closed_count`：关闭页面数量
  - `closed_urls`：被关闭页面的 URL 数组
- WS 调用：
  - 云端通过 WebSocket 下发 `local.http.post` 时，使用同样的 `path=/api/v1/page/close-by-url`
  - 请求体放在 `body` 中，例如：
```json
{
  "path": "/api/v1/page/close-by-url",
  "body": {
    "url_contains": "333333"
  }
}
```
- 统一元素定位对象说明：
  - `parent_classes`：可选，二维数组；每一项是一组父级 class 方案
  - `target_classes`：必填，二维数组；每一项是一组目标 class 方案
  - `find_attempts`：可选，查找重试次数，默认 `6`
  - `find_interval_ms`：可选，每次重试间隔毫秒数，默认 `500`
  - 查询规则：
    - 先主页面，再查所有 iframe
    - `parent_classes` 与 `target_classes` 按组做交叉查询
    - 每组内部的 class 按顺序兜底尝试
    - 命中即停，查不到才等待下次重试
    - `parent_classes` 可不传；不传时直接在当前范围内查 `target_classes`
  - 示例：
```json
{
  "parent_classes": [[".card-header"], [".header-wrap"]],
  "target_classes": [[".name"], [".candidate-name"]],
  "find_attempts": 6,
  "find_interval_ms": 500
}
```
- `POST /api/v1/page/scroll`
- 作用：滚动当前页面。
- 请求字段：`scroll_delay_min`, `scroll_delay_max`, `max_scrolls`, `element`
- `element`：使用上面的统一元素定位对象；传入后先把鼠标移动到目标元素上方再滚动
- `POST /api/v1/page/find-elements`
- 作用：查找一组元素，返回类似数组的结果，供云端逐个处理。
- 请求字段：
  - `element`（必填；统一元素定位对象）
  - `visible_only`（可选；默认 `true`，只返回当前可见元素）
  - `fields`（可选；字段提取配置）
- 返回字段：
  - `items`：数组，每项至少包含 `ref`、`index`；传入 `fields` 时额外包含 `fields`
  - `count`：找到的元素数量
- 说明：
  - `ref` 是本地程序生成的临时元素引用，后续仍可传给点击、滚动等接口
  - 推荐直接在 `find-elements` 中传入 `fields`，一次返回所有卡片字段，减少多次本地接口请求
- `fields` 格式示例：
```json
[
  {
    "name": {
      "parent_classes": [[".card-header"], [".header-wrap"]],
      "target_classes": [[".name"], [".candidate-name"]]
    }
  },
  {
    "age": {
      "parent_classes": [[".card-header"], [".header-wrap"]],
      "target_classes": [[".age"], [".candidate-age"]]
    }
  }
]
```
- 返回字段：
  - `fields`：如 `{"name":"张三","age":"26岁"}`
- 说明：
  - 如果传入 `element_ref`，则字段查找默认在该父级元素范围内进行
  - 每个字段自身也可再带 `parent_classes`；可传可不传
- `POST /api/v1/page/extract-text`
- 作用：提取目标元素的一整段文本。
- 请求字段：
  - `elements`（必填；统一元素定位对象数组，至少 1 项）
  - `mode`（可选；`dom` 或 `ocr`，默认 `dom`）
  - `delay_before`（可选；提取前等待秒数）
- 返回字段：
  - `text`：按顺序拼接后的整段文本（`\n\n` 连接）
  - `texts`：提取结果数组（与 `elements` 一一对应）
  - `matched`：命中选择器摘要（逗号拼接）
  - `matched_list`：每个元素的命中选择器摘要数组
- 说明：
  - `dom` 模式直接读取元素文本
  - `ocr` 模式会先对目标元素截图，再做本地 OCR 识别
  - 会按 `elements` 数组顺序逐个提取并返回 `texts`
- `POST /api/v1/page/in-viewport`
- 作用：判断某个元素当前是否位于可视区域内。
- 请求字段：
  - `element_ref`（可选；优先使用）
  - `element`（可选；统一元素定位对象）
- 返回字段：
  - `in_viewport`：布尔值
  - `matched`：命中的 `element_ref` 或选择器摘要
- 说明：
  - `element_ref` 与 `element` 二选一即可
  - 该接口只做判断，不会自动滚动
- `POST /api/v1/page/scroll-into-view`
- 作用：将某个元素滚动到当前可视区域内。
- 请求字段：
  - `element_ref`（可选；优先使用）
  - `element`（可选；统一元素定位对象）
- 返回字段：
  - `in_viewport`：布尔值
  - `matched`：命中的 `element_ref` 或选择器摘要
- 说明：
  - `element_ref` 与 `element` 二选一即可
  - 该接口只负责滚动到视口，不负责后续鼠标移动与点击
- `POST /api/v1/page/click`
- 作用：点击页面元素。
- 请求字段：`element`, `element_ref`(可选), `timeout`, `delay_before`
- `element`：使用上面的统一元素定位对象；传入后按协议定位目标元素，再把鼠标移动到元素中心并点击
- `element_ref`：可选；传入时表示先锁定某个父级元素，再在该父级元素内部查找并点击 `element`
- `POST /api/v1/page/press-key`
- 作用：向当前页面发送键盘按键（如关闭弹框）。
- 请求字段：
  - `key`：必填；按键名，例如 `Escape`、`Enter`、`ArrowDown`
- 返回字段：
  - `key`：实际发送的按键名
- `POST /api/v1/page/type-text`
- 作用：向当前已聚焦输入框分段输入文字，适合后续聊天输入场景。
- 请求字段：
  - `text`：必填；要输入的文字
  - `chunk_min`：可选；每段最少字符数，默认 `1`
  - `chunk_max`：可选；每段最多字符数，默认 `2`
  - `delay_min_ms`：可选；每段输入后的最小等待毫秒数，默认 `80`
  - `delay_max_ms`：可选；每段输入后的最大等待毫秒数，默认 `220`
- 请求示例：
  ```json
  {
    "text": "你好，我看到你的简历比较匹配，想和你沟通一下",
    "chunk_min": 1,
    "chunk_max": 2,
    "delay_min_ms": 80,
    "delay_max_ms": 220
  }
  ```
- 返回字段：
  - `chars`：输入字符数
  - `chunks`：实际分段次数
- 说明：
  - 该接口不查找输入框，也不主动点击输入框；调用前需要先让目标输入框获取焦点
  - 该接口通过键盘逐段输入，不使用剪贴板，也不是一次性粘贴
  - WebSocket 下发 `local.http.post` 时使用同样的 `path=/api/v1/page/type-text` 和请求字段
- `POST /api/v1/page/screenshot`
- 作用：按弹框选择器截图（支持滚动拼接），截图仅在内存中处理，不会长期写入本地文件。
- 请求字段：`modal_selectors`
- `GET /api/v1/page/url`
- 作用：获取当前页面 URL。
- `GET /api/v1/page/cookies`
- 作用：导出当前上下文 cookies。
- `POST /api/v1/page/load-profile`
- 作用：导入 profile 压缩包（base64）。
- 请求字段：`data`, `name`
- `POST /api/v1/page/export-profile`
- 作用：导出当前 profile 压缩包（base64）。

## Cookie 加密辅助

- `POST /api/v1/crypto/decrypt`
- 作用：解密云端返回的加密 cookie 数据。
- 请求字段：`encrypted_sk`, `encrypted_data`
- `POST /api/v1/cookies/decrypt`
- 作用：按当前机器码自动挑选密钥并直接解密出 cookie 列表，适合任务执行链路使用。
- 请求字段：`encrypted_data`, `encrypted_keys`

## OCR

- `GET /api/v1/ocr/status`
- 作用：查询 OCR 能力是否可用。
- `POST /api/v1/ocr/recognize`
- 作用：对图片执行 OCR。

## 提示音

- `POST /api/v1/sound/play`
- 作用：播放提示音一次。
- 请求字段：
  - `kind`：可选，`success` 或 `failed`（播放内置音频）
  - `url`：可选，网络音频地址；会先检查 `local-agent/assets/audio/` 是否已有同名文件，没有才下载再播放
- 说明：
  - 内置默认音频：
    - `success` -> `local-agent/assets/audio/success.mp3`
    - `failed` -> `local-agent/assets/audio/failed.mp3`
