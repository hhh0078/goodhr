# GoodHR5 Local Agent 接口清单

本文档整理 `goodhr5/local-agent` 当前对外 HTTP API，便于前端和云端联调。

## 通用说明

- 基础地址示例：`http://127.0.0.1:9001`
- 请求体格式：默认 `application/json`
- 成功返回：通常为 `{"ok": true, ...}`
- 失败返回：
  - 业务错误：`4xx` + JSON 错误信息
  - 未捕获异常：`500 Internal Server Error`

## 健康检查

- `GET /health`
- 作用：检查本地程序是否在线，返回机器码、公钥、端口、绑定账号。

## 云端账号与 WS

- `POST /api/v1/session/bind-cloud-user`
- 作用：绑定云端用户到本地程序。
- 请求字段：`cloud_user_id`, `cloud_email`, `agent_token`, `public_key`
- `POST /api/v1/ws/connect`
- 作用：本地主动连接云端 WebSocket。
- 请求字段：`cloud_ws_url`, `token`
- `GET /api/v1/ws/status`
- 作用：查询本地到云端 WS 的连接状态。
- `POST /api/v1/ws/disconnect`
- 作用：主动断开当前 WS 连接。
- `POST /api/v1/tasks/{task_id}/start-ws`
- 作用：建立任务级 WS，通知云端开始任务。
- 请求字段：`cloud_api_base`, `cloud_ws_url`, `token`
- `POST /api/v1/tasks/{task_id}/stop-ws`
- 作用：停止云端任务，并在无活跃任务时断开 WS。
- 请求字段：`cloud_api_base`, `token`

## Profile 管理

- `GET /api/v1/profiles`
- 作用：查询本地 profile 列表。
- 可选参数：`platform_id`
- `POST /api/v1/profiles`
- 作用：创建 profile 元数据。
- `DELETE /api/v1/profiles/{profile_id}`
- 作用：删除 profile 元数据。

## 本地任务数据

- `POST /api/v1/tasks/init`
- 作用：初始化本地任务目录。
- `GET /api/v1/tasks/{task_id}/candidates`
- 作用：读取本地候选人列表。
- `POST /api/v1/tasks/{task_id}/candidates`
- 作用：写入候选人记录。
- `DELETE /api/v1/tasks/{task_id}/candidates/{candidate_id}`
- 作用：删除候选人记录。
- `GET /api/v1/tasks/{task_id}/screenshots`
- 作用：查询截图列表。
- `GET /api/v1/tasks/{task_id}/screenshots/{filename}`
- 作用：下载单张截图。
- `DELETE /api/v1/tasks/{task_id}/screenshots/{filename}`
- 作用：删除单张截图。
- `POST /api/v1/tasks/{task_id}/ocr`
- 作用：保存 OCR 文本到本地任务目录。

## 浏览器控制

- `POST /api/v1/browser/start`
- 作用：启动浏览器。
- 请求字段：
  - `persistent`: 是否持久化
  - `user_data_dir`: 账号目录名（会映射到 `local-agent/cookies/{目录名}`）
  - `headless`: 是否无头
  - `humanize`: 是否仿人类
  - `proxy`: 代理地址
  - `cookies`: 可选，数组；启动后立即注入到当前浏览器上下文
  - `cookie_sync`: 可选；浏览器关闭后自动回传 cookie 到云端。字段：`platform_id`、`display_name`、`cloud_api_base`
- `POST /api/v1/browser/stop`
- 作用：关闭浏览器。
- `GET /api/v1/browser/status`
- 作用：查询浏览器运行状态。

## 页面操作

- `POST /api/v1/page/open`
- 作用：打开指定 URL 并注册为默认页面。
- 请求字段：`url`, `timeout`, `user_data_dir`(可选), `persistent`(可选), `headless`(可选), `humanize`(可选), `proxy`(可选), `cookies`(可选，数组，传入时先注入再导航), `cookie_sync`(可选，同 browser/start)
- 注意：必须先调用 `/api/v1/browser/start`；未启动会返回 `400`。
  - 如果传入 `user_data_dir`，接口会自动按该账号目录切换浏览器再打开页面，不需要先单独调用 `browser/start`。
- `POST /api/v1/page/scroll`
- 作用：滚动当前页面。
- 请求字段：`scroll_delay_min`, `scroll_delay_max`, `max_scrolls`
- `POST /api/v1/page/extract`
- 作用：按选择器提取页面字段。
- 请求字段：
  - `selectors`（必填）
  - `mode`（`single`/`batch`）
  - `card_selector`（`batch` 模式用）
- `POST /api/v1/page/click`
- 作用：点击页面元素。
- 请求字段：`selector`, `timeout`, `delay_before`
- `POST /api/v1/page/screenshot`
- 作用：按弹框选择器截图（支持滚动拼接）。
- 请求字段：`modal_selectors`
- `GET /api/v1/page/url`
- 作用：获取当前页面 URL。
- `GET /api/v1/page/cookies`
- 作用：导出当前上下文 cookies。
- `POST /api/v1/page/load-profile`
- 作用：导入 profile 压缩包（base64）。
- 请求字段：`data`, `name`
- `POST /api/v1/page/export-profile`
- 作用：导出当前 profile 压缩包（base64）。

## Cookie 加密辅助

- `POST /api/v1/crypto/decrypt`
- 作用：解密云端返回的加密 cookie 数据。
- 请求字段：`encrypted_sk`, `encrypted_data`

## OCR

- `GET /api/v1/ocr/status`
- 作用：查询 OCR 能力是否可用。
- `POST /api/v1/ocr/recognize`
- 作用：对图片执行 OCR。

## 提示音

- `POST /api/v1/sound/play`
- 作用：播放提示音一次。
- 请求字段：
  - `kind`：可选，`success` 或 `failed`（播放内置音频）
  - `url`：可选，网络音频地址；会先检查 `local-agent/assets/audio/` 是否已有同名文件，没有才下载再播放
- 说明：
  - 内置默认音频：
    - `success` -> `local-agent/assets/audio/success.mp3`
    - `failed` -> `local-agent/assets/audio/failed.mp3`
